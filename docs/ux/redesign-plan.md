# תוכנית עבודה — SmartGrader: עיצוב מחדש הוליסטי "Warm Minimal"

> גרסה 4 · 13.07.2026 · משימה זו = צד לקוח (עיצוב ואפיון) בלבד

## מטרה

הפיכת SmartGrader למוצר אמיתי לשימוש: שפה עיצובית אחת נקייה ונעימה על בסיס הפלטה החמה הקיימת, בנייה מחדש של תוכן המסכים כך שכל מסך יציג רק מה שצריך ונוח, וסגירת פערים פונקציונליים.

## החזון העיצובי — "מינימליזם חם"

הפלטה הקיימת נשארת הזהות: רקע בז' `#f3efe7`, משטחים `#fbfaf8`, accent טאופה `#8a6a54`, פונט Rubik.

1. **היררכיה ויזואלית של 3 רמות בלבד** — רקע שטוח ← כרטיס ← שכבת-על (דיאלוג/תפריט). בלי ערימת צללים ומסגרות.
2. **גריד 8pt + סקאלה טיפוגרפית אחת** — display / title / body / caption. כל מסך בנוי מאותם אבני בניין.
3. **שלד אפליקציה חדש** — topbar דק; תבנית "כותרת עמוד" אחידה (כותרת + תת-כותרת + פעולה ראשית + breadcrumb); footer מינימלי; ה-Hero הענק (300px) הופך לרכיב קבלת פנים קומפקטי ב-Dashboard בלבד.
4. **ארכיטקטורת מידע ברורה** — Dashboard = מרכז תובנות (ממוצעים, מגמות, מה דורש טיפול) ← צלילה: שיעורים ← תרגילים ← הגשות; סטודנטים ← הגשות. Breadcrumbs עקביים.
5. **שלוש תבניות-אם** — תבנית רשימה, תבנית טופס, תבנית פירוט. כל מסך הוא מופע של אחת מהן.
6. **מערכת סטטוסים סמנטית בגוני הפלטה** — מרווה=הצלחה, ענבר=בתהליך, טרקוטה=שגיאה; אייקון ליד כל צבע (נגישות).
7. **אזור תלמידה — "המסע שלי"** (אפיון+עיצוב בלבד) — השיעורים שלי ← הגשת קוד ← פידבק AI וציון; מסך כניסה בעיצוב תואם.

## שלב 0 — מסמך אפיון-על (`docs/ux/master-spec.md`)

- מטרת המוצר, פרסונות (מ-personas.md), מפת מסכים: אזור מורה / אזור תלמידה / מסך כניסה.
- תכולת אזור תלמידה: צפייה בהגשות ובציונים שלה, הגשת קוד לתרגיל, רשימת שיעורים ותרגילים, פידבק AI.
- דפוסים משותפים: מצבי ריק/טעינה/שגיאה, טוסטים, אישורי מחיקה, תבנית טבלה (בחירה מרובה + תפריט פעולות), פורמט תאריך אחיד `13.07.26 10:33`, סמנטיקת צבעי סטטוס.
- אפיון התראות (עתידי): הפעמון יציג הגשות שסיימו בדיקת AI; כרגע — הסרת המונה המזויף "3".

## שלב 1 — תשתית עיצובית (`client/src/styles.css`)

1. הוספת טוקנים לפי `client/spec.md`: `--radius-sm/md/lg`, `--shadow-sm/md`, `--space-1..6`, `--text-sm/base/lg/xl`. הצבעים הקיימים נשארים.
2. **שדות טופס**: הקטנת גבהים (44px ← ~38px), textarea בגודל הגיוני, מסגרת דקה ועדינה (`--app-border`) במקום המסגרת החומה העבה, focus ring רך.
3. **צבעי סטטוס** מותאמים לפלטה: `--status-success` (מרווה), `--status-warn` (ענבר), `--status-error` (טרקוטה עמומה), `--status-info` — במקום אדום/כחול צורמים.
4. **Paginator**: עיצוב מחדש + תיקון כיווני חצים ב-RTL (ראשון/קודם/הבא/אחרון).
5. **Audit אייקונים הפוכים** ב-RTL: חצי חזרה, chevrons.
6. **תאריכים**: רישום locale עברי ב-`app.config.ts` + פורמט אחיד בכל המערכת (בלי "AM", בלי אייקון לוח שנה).
7. כרטיסים/צללים/רדיוסים אחידים; צמצום Hero; **פישוט Footer** (הסרת ציטוט/חיפוש/דירוג כוכבים — נשארים מותג + קישורים).
8. **Topbar**: הסרת מונה הפעמון המזויף, סידור אווטאר + שם משתמש.

## שלב 2 — תבנית רשימה אחידה (כל הטבלאות)

9. **בחירה מרובה — עיצוב בלבד**: עמודת checkbox + סרגל בחירה ("נבחרו 3 · מחיקת נבחרים") **ללא לוגיקת מחיקה בפועל** (מימוש במשימה עתידית).
10. **עמודת פעולות ← תפריט נפתח (⋯)**: עריכה ומחיקה בתוך התפריט; **"צפייה בהגשות" כאייקון נפרד** מחוץ לתפריט.
11. מיקום כפתורים לפי UX: פעולה ראשית (+ חדש) בראש בצד ההתחלה, חיפוש וסינון בשורה אחת, ייצוא ליד הטבלה — עקבי בכל המסכים.
12. עיצוב מחדש של פאנל הסינון ("מיון לפי"): ריווח, יישור, כפתור איפוס מעוצב.

## שלב 3 — בנייה מחדש של המסכים + מידע מועיל

- **Dashboard**: מיקוד KPI + ממוצעים (ממוצע כיתה, ממוצע לשיעור); צמצום טבלת ההגשות הכפולה ← קישור למסך הגשות.
- **Students רשימה**: פאנל סינון מתקפל; איחוד עמודות המונים לעמודת "פעילות"; הוספת עמודת ממוצע ציונים לתלמיד.
- **Students טופס**: placeholders בעברית (במקום "e.g., Noah Cohen"), סימון שדות חובה.
- **Lessons**: הסרת עמודת "שם מורה" (תמיד זהה); הוספת ממוצע ציון לשיעור; כפתור מספר תרגילים ← פעולה ברורה.
- **Assignments**: עברית מלאה + הסרת עמודות מתות (לפי assignments-flow.md); סטטיסטיקת הגשות.
- **Submissions רשימה**: הסרת תצוגת קוד מוטבעת (רק בפירוט); תא תאריך נקי; פעולות לפי התבנית החדשה.
- **Submission פירוט**: איחוד קופסאות שגיאה/הערות לאזור סטטוס אחד עקבי.
- **LessonResults**: פרוגרס סטנדרטי ("2/5") במקום תצוגת השבר; טיפול בכפתור Finalize (חיבור בשלב 4 או הסרה).
- **מסך כניסה + מסכי תלמידה**: עיצוב ואפיון בלבד (ב-master-spec.md), ללא לוגיקת auth.
- כל מסך: אימות RTL + רוחבים 360/768/1280.

## שלב 4 — פערים פונקציונליים

1. שגיאות inline בטופס Submissions (לפי הדפוס של assignment-form.component.ts).
2. אימות/השלמת polling כל 7 שניות כשהסטטוס ProcessingAi.
3. **LessonResults שרת**: הפעלת ה-endpoint המוער `POST /api/lesson-results/complete` ב-LessonResultController.cs + LessonResultProfile (AutoMapper), וחיבור ה-frontend.

## שלב 5 — ניקיון ואימות

1. מחיקת קבצים כפולים ישנים (lessons.component.ts, students.component.ts) אחרי אימות שאינם ב-routes.
2. Audit נגישות מול docs/ux/accessibility-checklist.md.
3. `ng build` נקי; מעבר ידני על כל מסך ב-3 רוחבים + RTL; זרימות יצירה/עריכה/מחיקה עם טוסטים בעברית; LessonResults מקצה לקצה.

## החלטות

- הצבעים והפונט נשארים — משתנה השפה: היררכיה, ריווח, תבניות אחידות.
- בחירה מרובה = UI בלבד, בלי מחיקה בפועל.
- התחברות אמיתית + מסכי תלמידה פונקציונליים = משימה נפרדת; כאן אפיון ועיצוב בלבד.
- מחיקה מרובה בשרת (bulk endpoint) — נדחה לעתיד.
- פעמון: הסרת המונה המזויף עכשיו; התראות אמיתיות לפי האפיון בהמשך.
